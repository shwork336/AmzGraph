# 竞品数据基础模块实现计划

> **面向 AI 代理的工作者：** 建议使用 subagent-driven-development 或 executing-plans 按任务执行此计划。步骤使用复选框语法跟踪进度。涉及业务规则的 Java 文件必须按本计划中的 review 关卡逐个交给用户确认。

**目标：** 建立可追溯的竞品快照模型、外部数据源端口、手工批量补录、完整历史查询和每个 ASIN 最新快照查询。

**架构：** 新增独立 `competitor` 模块。`CompetitorSnapshotService` 读取 Listing 任务状态和原始 ASIN 范围，但不修改任务或 Brief；手工快照采用追加写入，latest 视图由应用服务聚合。

**技术栈：** Java 17、Spring Boot 4、Spring Web MVC、Spring Data JPA、Bean Validation、Jackson 3、H2、JUnit 5、AssertJ、MockMvc。

**设计依据：** `docs/plans/竞品数据基础模块设计.md`

### 1.3 文件职责

- `CompetitorSourceType`：稳定表达手工、Bright Data 和缓存来源。
- `CompetitorSnapshot`：持久化标准化竞品数据和采集审计信息。
- `CompetitorSnapshotRepository`：查询任务快照稳定倒序历史。
- `CompetitorSnapshotData`：外部供应商标准化输出，不与 JPA 实体耦合。
- `CompetitorDataProvider`：后续 Bright Data 适配器依赖的业务端口。
- `CompetitorSnapshotService`：执行状态、ASIN 范围、批量原子性和 latest 聚合规则。
- `ManualCompetitorSnapshotRequest`：单条手工快照输入。
- `SubmitManualCompetitorsRequest`：批量补录操作者和快照集合。
- `CompetitorSnapshotResponse`：稳定 API 输出，不暴露实体。
- `ListingTaskController`：增加三个薄竞品接口。

@Transactional(readOnly = true)
public List<CompetitorSnapshotResponse> listLatestSnapshots(String taskId)

算法：

1. 读取任务原始 ASIN 顺序并标准化。
2. 读取已按时间倒序的全部快照。

4. 按任务原始 ASIN 顺序取出存在的最新快照。
5. 不为没有快照的 ASIN 创建空记录。
