# 导出列表筛选最小闭环实现计划

> 本计划承接导出列表查询最小闭环。
> 本阶段目标是在任务导出历史列表上补充格式和状态筛选，便于前端归档导出页快速展示失败记录、某一种格式的最新导出记录，或排查重试结果。

## 1. 当前基线

已完成：

- 已完成任务可导出 ZIP、Markdown、Excel、Word。
- 四种导出格式都支持失败重试。
- 单个导出包可查询。
- 任务下全部导出记录可通过 `GET /api/v1/listing/{taskId}/exports` 查询。
- 列表按创建时间倒序、导出包 ID 倒序返回。

当前未完成：

- 导出列表不能按格式筛选。
- 导出列表不能按状态筛选。
- 前端归档导出页需要自己过滤失败记录或某种格式记录。

## 2. 阶段目标

本阶段完成后，应满足：

- `GET /api/v1/listing/{taskId}/exports` 保持兼容。
- 支持可选查询参数：
  - `format`：`ZIP`、`MARKDOWN`、`EXCEL`、`WORD`
  - `status`：导出状态字符串，例如 `SUCCEEDED`、`FAILED`
- 未传筛选参数时仍返回全部导出记录。
- 同时传 `format` 和 `status` 时取交集。
- `format` 非法返回 `INVALID_REQUEST`。
- 空白 `format` / `status` 按未传处理。
- 任务不存在返回 `TASK_NOT_FOUND`。
- 查询接口不触发导出、重试或任务状态变更。
- 完整测试通过。

## 3. 范围边界

### 3.1 本阶段做

- 扩展 `ExportPackageService`：
  - 将 `listExportPackages(String taskId)` 扩展为支持 `format`、`status`
  - 保留旧方法作为无筛选重载，减少已有调用影响
  - 使用内存过滤当前任务的导出历史
- 扩展 `ListingTaskController`：
  - `GET /api/v1/listing/{taskId}/exports?format=ZIP&status=FAILED`
- 补充服务测试：
  - 按格式筛选
  - 按状态筛选
  - 同时按格式和状态筛选
  - 空白参数等同未传
  - 非法格式返回 `INVALID_REQUEST`
- 补充 Controller 测试：
  - query 参数传递成功
  - 非法格式错误透传

### 3.2 本阶段不做

- 不做分页。
- 不做时间范围筛选。
- 不做多格式筛选。
- 不做模糊搜索。
- 不新增仓储查询方法。
- 不改造为异步导出任务列表。

## 4. 设计原则

- 保持原列表接口路径不变，只增加可选 query 参数。
- 当前阶段导出记录量小，先用服务层内存过滤；后续分页和数据量上来后再下推到仓储。
- `format` 使用 `ExportFormat.valueOf` 校验，非法值返回稳定业务错误。
- `status` 保持字符串，兼容当前 `ExportPackage.status` 字段。
