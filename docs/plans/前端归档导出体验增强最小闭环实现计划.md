# 前端归档导出体验增强最小闭环实现计划

> 本计划承接前端竞品数据页最小闭环。目标是在已有归档导出页基础上补齐导出记录可读性，让运营能快速判断导出状态、打开文件链接、查看 manifest、取消信息和导出时间线。

## 1. 当前基线

已完成：

- 前端归档导出页 `/tasks/:taskId/export`。
- 后端导出响应包含 `fileUrl`、`manifestUrl`、`failureReason`、`cancelReason`、`canceledBy`、`canceledAt`、`createdAt`、`startedAt`、`updatedAt`。
- 前端已支持创建、运行、取消、重试导出记录。
- 前端已展示任务内审计日志。

当前不足：

- 导出文件和 manifest 只是表格文本，不便打开。
- 归档页没有导出状态摘要。
- 取消人、取消时间和时间线字段没有集中展示。
- 表格中导出记录缺少详情查看区。

## 2. 阶段目标

- 归档导出页展示状态摘要：
  - 总数。
  - PENDING。
  - RUNNING。
  - SUCCEEDED。
  - FAILED。
  - CANCELED。
- 导出历史表支持：
  - 文件链接。
  - manifest 链接。
  - 详情选择。
- 导出详情展示：
  - 导出包 ID。
  - 格式和状态。
  - 文件/manifest。
  - 失败或取消原因。
  - 取消人和取消时间。
  - 创建、开始、更新时间。
  - 包含资产 ID。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端归档导出页展示增强。
- 不改变后端导出状态机。
- 不新增后端接口。

本阶段不做：

- 不实现真实文件下载代理。
- 不实现批量导出操作。
- 不实现导出文件预览。
- 不修改导出审计后端。
