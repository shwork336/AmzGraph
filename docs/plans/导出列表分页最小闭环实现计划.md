# 导出列表分页最小闭环实现计划

> 本计划承接导出列表查询和导出列表筛选最小闭环。
> 本阶段目标是在不破坏现有列表接口的前提下，为前端归档导出页提供分页元数据，避免导出历史增长后一次性返回过多记录。

## 1. 当前基线

已完成：

- `GET /api/v1/listing/{taskId}/exports` 可返回任务下全部导出记录。
- 导出列表支持 `format` 和 `status` 筛选。
- 无筛选参数时返回全部导出记录。
- 列表按创建时间倒序、导出包 ID 倒序返回。

当前未完成：

- 导出列表没有分页参数。
- 导出列表响应没有总数、页码、页大小和是否有下一页等元数据。
- 前端归档导出页只能自行截断或全量加载。

## 2. 阶段目标

本阶段完成后，应满足：

- 保持原接口兼容：
  - 不传 `page` / `size` 时，仍返回 `List<ExportPackageResponse>`。
- 支持分页查询：
  - `GET /api/v1/listing/{taskId}/exports?page=0&size=20`
  - 返回分页响应对象。
- 分页响应包含：
  - `items`
  - `page`
  - `size`
  - `totalItems`
  - `totalPages`
  - `hasNext`
  - `hasPrevious`
- 分页与 `format`、`status` 筛选可组合。
- `page < 0` 返回 `INVALID_REQUEST`。
- `size <= 0` 或 `size > 100` 返回 `INVALID_REQUEST`。
- 任务不存在返回 `TASK_NOT_FOUND`。
- 完整测试通过。

## 3. 范围边界

### 3.1 本阶段做

- 新增分页响应 DTO：
  - `PagedResponse<T>`
- 扩展 `ExportPackageService`：
  - 新增 `listExportPackagesPage(String taskId, String format, String status, int page, int size)`
  - 复用当前筛选逻辑
  - 在服务层做分页截取
- 扩展 `ListingTaskController`：
  - `page` 和 `size` 同时传入时返回分页响应
  - 未传分页参数时保持返回列表
  - 只传一个分页参数时返回 `INVALID_REQUEST`
- 补充服务测试：
  - 第一页分页
  - 第二页分页
  - 分页与筛选组合
  - 空结果分页
  - 非法分页参数
- 补充 Controller 测试：
  - 无分页参数仍返回数组
  - 有分页参数返回分页对象
  - 只传一个分页参数返回 `INVALID_REQUEST`

### 3.2 本阶段不做

- 不新增仓储分页查询。
- 不做排序字段自定义。
- 不做游标分页。
- 不做时间范围筛选。
- 不改变已有无分页响应结构。

## 4. 设计原则

- 兼容优先：已有列表调用不需要改。
- 当前阶段数据量小，先复用服务层筛选和内存分页；后续记录增长后再下推到数据库分页。
- 分页参数显式成对出现，避免前端误传导致响应结构不确定。
- 最大 `size` 限制为 100，防止归档页误请求过大列表。
