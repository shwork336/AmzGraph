# 前端路由按需拆包优化最小闭环实现计划

> 本计划承接前端错误处理和空状态统一最小闭环。目标是降低前端主包体积，消除 Vite 构建中的大 chunk 警告，让多页面工作台按路由加载。

## 1. 当前基线

已完成：

- 前端工作台已实现任务列表、创建任务、任务详情、竞品、Brief、文案、图片、合规、终审、导出和审计页面。
- 当前构建通过。

当前不足：

- `router.ts` 静态导入所有页面。
- Vite 构建持续提示 chunk 超过 500 kB。
- Naive UI 和 Vue 运行时未显式 vendor 分包。

## 2. 阶段目标

- 路由页面改为动态 import。
- Vite 配置拆出：
  - `vue-vendor`
  - `naive-ui`
- 前端构建通过，并不再出现 500 kB 大 chunk 警告。
- 后端测试通过。

## 3. 范围边界

本阶段做：

- 前端路由级懒加载。
- Vite manualChunks 配置。

本阶段不做：

- 不引入新的构建插件。
- 不重构 Naive UI 按组件自动导入。
- 不修改业务页面逻辑。
