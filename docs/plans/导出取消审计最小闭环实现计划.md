# 导出取消审计最小闭环实现计划

> 本计划承接导出时间字段增强最小闭环。目标是在取消待执行导出任务时记录取消人、取消原因和取消时间，让运营侧取消动作可追踪、可回放。

## 1. 当前基线

已完成：

- 导出记录支持 `PENDING -> CANCELED`。
- 取消只允许发生在 `PENDING` 状态。
- 取消后保留导出记录，不生成文件。
- 导出记录已有 `createdAt`、`startedAt`、`updatedAt`。
- 导出列表可返回 `CANCELED` 记录。

当前不足：

- 取消记录没有取消人。
- 取消记录没有取消原因。
- 取消记录没有独立取消时间。
- 前端归档导出页后续无法展示取消审计信息。

## 2. 阶段目标

- `ExportPackage` 新增：
  - `canceledBy`
  - `cancelReason`
  - `canceledAt`
- 取消接口接收请求体：
  - `canceledBy` 必填。
  - `cancelReason` 必填。
- 成功取消时写入取消审计字段并更新 `updatedAt`。
- `ExportPackageResponse` 返回取消审计字段。
- 导出列表和取消响应都能看到取消审计信息。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 修改导出实体字段。
- 新增取消请求 DTO。
- 修改取消服务方法签名。
- 修改 Controller 取消接口请求体。
- 修改响应 DTO。
- 补充服务层和 Controller 测试。

本阶段不做：

- 不取消 `RUNNING` 记录。
- 不删除已生成文件。
- 不新增权限系统或真实当前用户上下文。
- 不新增数据库 migration 文件。
- 不改变取消接口路径。
