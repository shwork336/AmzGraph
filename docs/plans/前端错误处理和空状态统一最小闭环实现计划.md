# 前端错误处理和空状态统一最小闭环实现计划

> 本计划承接前端工作台导航信息架构整理最小闭环。目标是收敛前端页面中重复的时间格式化、空值展示和错误消息处理，让高频页面的展示行为保持一致。

## 1. 当前基线

已完成：

- 前端已实现任务列表、任务详情、归档导出、操作审计等核心页面。
- 各页面已有基础加载态和错误提示。

当前不足：

- 多个页面重复实现 `displayTime`。
- 多个页面重复使用 `error instanceof Error ? error.message : fallback`。
- 表格空状态文案不统一。

## 2. 阶段目标

- 新增共享展示工具：
  - `displayTime`
  - `displayValue`
  - `errorMessage`
- 高频页面使用共享工具：
  - 任务列表页。
  - 任务详情页。
  - 归档导出页。
  - 操作审计页。
- 高频表格补齐统一空状态文案。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端展示工具抽取。
- 高频页面迁移。
- 表格空状态文案补齐。

本阶段不做：

- 不引入全局错误边界。
- 不重构所有页面。
- 不改变 API 响应处理协议。
- 不新增后端接口。
