# 前端任务路由路径工具统一最小闭环实现计划

> 本计划承接前端未知路由兜底页。目标是把任务详情及任务子流程页面的路径拼接收敛为共享工具，避免各页面重复手写 `/tasks/${encodeURIComponent(taskId)}/...`。

## 1. 当前基线

已完成：

- 任务详情页和各任务子页面已注册路由。
- 任务子页面已经完成浏览器冒烟。
- 未知路径已有兜底页。

当前不足：

- 多个页面重复拼接任务路径。
- `encodeURIComponent(taskId)` 分散在页面里。
- 后续新增任务子流程时容易漏编码或路径不一致。

## 2. 阶段目标

- 新增共享任务路径工具。
- 任务列表、任务详情、任务创建、导出和子页面返回详情使用共享路径。
- 保持现有路由结构和跳转行为不变。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 新增 `frontend/src/utils/taskRoutes.ts`。
- 替换页面内重复路径拼接。
- 不修改 `router.ts` 的路由定义。

本阶段不做：

- 不引入命名路由。
- 不改变任务子页面 URL 结构。
- 不调整后端 API 路径。

## 4. 任务拆解

- [x] 新增共享任务路径工具。
- [x] 任务列表页和创建任务页使用共享路径。
- [x] 任务详情页工作流入口使用共享路径。
- [x] 任务子页面返回详情使用共享路径。
- [x] 归档导出页打开任务使用共享路径。
- [x] 前端 `npm.cmd run build` 通过。
- [x] 后端 `.\gradlew.bat test --console=plain` 通过。

## 5. 验收清单

- [x] 页面中不再散落任务路径编码拼接。
- [x] 现有任务详情和子页面 URL 保持不变。
- [x] 前端构建通过。
- [x] 后端完整测试通过。
