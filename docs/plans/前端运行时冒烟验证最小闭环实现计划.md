# 前端运行时冒烟验证最小闭环实现计划

> 本计划承接前端提交前整体回归最小闭环。目标是在构建和后端测试之外，用真实浏览器打开前端工作台关键路由，确认页面能完成 Vue 运行时挂载、基础导航和错误态展示。

## 1. 当前基线

已完成：

- 前端任务列表、创建任务、任务详情。
- 前端竞品、Brief、文案、图片、合规、终审、归档导出、操作审计页面。
- 顶部导航、任务详情工作流入口、错误和空状态工具函数。
- 路由按需拆包和 Vite 分包优化。
- 前端构建通过，后端完整测试通过。

当前不足：

- 之前主要依赖类型检查和构建，缺少真实浏览器运行时验证。
- 多个页面使用 `useMessage()`，根组件需要提供 Naive UI 的消息上下文。
- 需要确认关键路由在后端不可用或无数据时仍能显示页面结构和错误态。

## 2. 阶段目标

- 前端开发服务可启动。
- `/tasks`、`/tasks/new`、`/audit-logs` 至少能在浏览器中挂载页面内容。
- 页面运行时无阻断性控制台错误。
- 发现的运行时基础设施问题同步修复。

## 3. 范围边界

本阶段做：

- 根级 UI provider 补齐。
- 浏览器打开关键一级路由做冒烟验证。
- 前端构建回归。

本阶段不做：

- 不补后端种子数据。
- 不做完整端到端业务流。
- 不新增新的业务页面。

### 任务 1：运行时基础设施

- [x] 打开 `/tasks` 并确认任务列表页面内容可见。
- [x] 打开 `/tasks/new` 并确认创建任务页面内容可见。
- [x] 打开 `/audit-logs` 并确认操作审计页面内容可见。

- [x] 前端 `npm.cmd run build` 通过。
- [x] 后端 `.\gradlew.bat test --console=plain` 通过。

- [x] 页面无缺失 provider 导致的运行时挂载失败。
- [x] 前端构建通过。
- [x] 后端完整测试通过。
