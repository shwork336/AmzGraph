# 导出任务异步化最小闭环实现计划

> 本计划承接导出列表分页最小闭环。目标是在不破坏现有同步导出接口的前提下，先拆出“创建导出记录”和“执行导出记录”的状态边界，为后续接入后台队列或调度器做准备。

## 1. 当前基线

已完成：

- ZIP、Markdown、Excel、Word 四种导出格式。
- 导出失败重试。
- 导出列表查询、筛选、分页。
- 导出记录已有 `PENDING`、`RUNNING`、`SUCCEEDED`、`FAILED` 的状态字段语义。

当前不足：

- 导出接口仍在请求线程里同步生成文件。
- 前端无法先创建导出任务，再通过列表或详情查看状态。
- 后续接队列前缺少稳定的 worker 执行入口。

## 2. 阶段目标

- 保留现有同步导出接口：
  - `POST /api/v1/listing/{taskId}/export`
  - `POST /api/v1/listing/{taskId}/export/markdown`
  - `POST /api/v1/listing/{taskId}/export/excel`
  - `POST /api/v1/listing/{taskId}/export/word`
- 新增创建待执行导出记录接口：
  - `POST /api/v1/listing/{taskId}/exports?format=ZIP`
  - `format` 为空时默认 `ZIP`
  - 返回 `PENDING`
- 新增执行待处理导出接口：
  - `POST /api/v1/listing/export/{exportPackageId}/run`
  - 只允许执行 `PENDING` 记录
  - 执行成功后返回 `SUCCEEDED`
  - 执行失败后记录 `FAILED` 和失败原因
- 执行时校验待执行记录仍指向任务当前最终图文选择。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 服务层拆分 `createPendingExportPackage`。
- 服务层拆分 `runPendingExportPackage`。
- 同步导出方法复用统一导出执行逻辑。
- Controller 新增创建与执行接口。
- 服务层和 Controller 测试覆盖。

本阶段不做：

- 不引入消息队列。
- 不引入 Spring `@Async` 或线程池配置。
- 不新增数据库分页查询优化。
- 不改变现有同步导出接口响应结构。
- 不改变失败重试创建新记录的审计策略。
