# 图片生成最小闭环实现计划

> 本计划承接已完成的文案生成最小闭环。
> 本阶段目标是让任务在 Brief 已批准后可以生成一组可查询的图片版本和图片资产，先用端口和占位实现跑通状态机、版本保存、资产保存和查询接口，暂不接入真实图片模型。

## 1. 当前基线

已完成：

- Listing 任务提交、Brief 人工审核、竞品手工快照基础模块。
- 文案生成最小闭环：
  - `TextVersionRepository`
  - `ListingTextGenerator`
  - `PlaceholderListingTextGenerator`
  - `TextGenerationService`
  - `TextVersionResponse`
  - `POST /api/v1/listing/{taskId}/versions/text/generate`
  - `GET /api/v1/listing/{taskId}/versions/text`
- `ImageVersion`、`ImageAsset`、`ImageAssetType` 实体和枚举骨架已存在。
- `ImageAssetRepository` 仅有基础 `JpaRepository` 能力。
- `CAR_STEREO` 模板已配置默认图片资产类型和尺寸档位。
- 当前完整测试 `.\gradlew.bat test` 已通过。

当前未完成：

- `ImageVersionRepository`。
- `ImageAssetRepository` 按图片版本查询与排序。
- 图片生成端口和占位实现。
- 图片版本响应 DTO、图片资产响应 DTO。
- 图片生成应用服务。
- 图片版本和资产查询接口。
- 图片生成成功后的 `imageStatus` 状态流转。
- 文案和图片均完成后推进 `WAIT_FINAL_APPROVE`。
- 真实图片模型、合规检查、终审选择和导出。

## 2. 阶段目标

本阶段完成后，应满足：

- 系统存在稳定的 `ImageVersionRepository`。
- `ImageAssetRepository` 支持按 `imageVersionId` 查询资产并按 `sortOrder + assetId` 排序。
- 业务层只依赖 `ImageAssetGenerator` 端口，不直接依赖具体图片模型 SDK。
- 占位图片生成器可以基于已批准 Brief 和模板生成一组 `ImageAsset`。
- 任务在 `GENERATING` 状态下可生成图片版本。
- 图片生成成功后，`ListingTask.imageStatus` 从 `NOT_STARTED` 或 `RUNNING` 推进到 `SUCCEEDED`。
- 图片生成成功后主任务仍保持 `GENERATING`，暂不进入 `WAIT_FINAL_APPROVE`。
- 可以查询任务下全部图片版本，按创建时间和版本 ID 倒序返回。
- 可以查询指定图片版本下的全部图片资产，按 `sortOrder + assetId` 排序返回。
- 完整测试通过。

### 3.2 本阶段不做

- 不推进 `WAIT_FINAL_APPROVE`。
- 不做导出。
- 不引入异步调度。

## 4. 设计原则

- 业务服务通过 `ImageAssetGenerator` 端口调用图片生成能力，便于后续替换真实适配器。
- `ImageVersion` 表达一次图片组生成，`ImageAsset` 表达组内单张图片。
- 本阶段每次生成追加一个新的 `ImageVersion`，不覆盖历史版本。
- 默认图片资产类型从 `CategoryTemplate.defaultImageAssetTypesJson` 读取，不在服务里硬编码。
- 尺寸信息从 `CategoryTemplate.sizeProfilesJson` 读取，占位实现按类型选择尺寸档位。
- 图片生成失败只影响 `imageStatus` 和当前图片版本状态，不回退主任务状态。
- API DTO 不直接暴露 JPA 实体。

- [x] **步骤 4：实现 `ImageGenerationService`**

职责：

- 查询任务。
- 校验任务主状态和 Brief 状态。
- 获取最新已批准 Brief。
- 获取类目模板。
- 设置 `imageStatus = RUNNING`。
- 创建 `ImageVersion`，状态先设为 `RUNNING`。
- 调用 `ImageAssetGenerator`。
- 补齐每个 `ImageAsset.assetId`、`imageVersionId`、`sortOrder`。
- 保存 `ImageVersion` 和 `ImageAsset`。
- 成功后设置 `ImageVersion.status = SUCCEEDED`、`imageStatus = SUCCEEDED`。
- 查询图片版本列表。
- 查询指定图片版本资产，要求图片版本属于任务。
