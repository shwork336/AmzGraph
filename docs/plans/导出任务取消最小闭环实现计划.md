# 导出任务取消最小闭环实现计划

> 本计划承接后台导出 Worker 最小闭环。目标是在 worker 扫描 `PENDING` 记录之前，允许运营取消误创建或不再需要的导出任务，避免后台继续生成无用交付文件。

## 1. 当前基线

已完成：

- 创建待执行导出记录：`PENDING`。
- 显式执行待处理导出：`PENDING -> RUNNING -> SUCCEEDED/FAILED`。
- 后台 worker 可扫描并处理 `PENDING` 记录。
- 导出列表支持按 `status` 筛选。

当前不足：

- 已创建但尚未执行的导出任务不能取消。
- 误创建导出后只能等待 worker 执行，产生不必要文件。
- 前端归档导出页缺少取消操作的后端边界。

## 2. 阶段目标

- 新增取消待执行导出服务能力：
  - 只允许取消 `PENDING` 导出记录。
  - 取消后状态更新为 `CANCELED`。
  - 取消不删除审计记录。
  - 取消不生成文件。
- 新增 HTTP 接口：
  - `POST /api/v1/listing/export/{exportPackageId}/cancel`
- 导出列表仍可返回 `CANCELED` 记录，并可通过 `status=CANCELED` 筛选。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 扩展 `ExportPackageService`。
- 扩展 `ListingTaskController`。
- 补充服务层和 Controller 测试。
- 更新计划验收状态。

本阶段不做：

- 不取消 `RUNNING` 记录。
- 不删除已生成文件。
- 不新增取消人、取消原因字段。
- 不处理 worker 并发领取锁。
- 不新增前端页面。
