# 前端创建任务页最小闭环实现计划

> 本计划承接前端任务列表页最小闭环。目标是让运营可以通过前端创建 Listing 任务，并在提交成功后进入归档导出或任务列表。

## 1. 当前基线

已完成：

- 前端任务列表页 `/tasks`。
- 前端归档导出页 `/tasks/:taskId/export`。
- 后端任务提交接口 `POST /api/v1/listing/submit`。
- 后端限制 Markdown 产品资料必填、产品图数量 1-4、产品图 MIME 必须为 `image/*`。

当前不足：

- 前端没有创建任务页。
- 运营仍需用外部工具提交 multipart 表单。
- 前端缺少与后端一致的上传校验。

## 2. 阶段目标

- 新增路由：
  - `/tasks/new`
- 任务列表页提供创建任务入口。
- 创建任务页支持：
  - 上传 Markdown 产品资料。
  - 上传 1-4 张产品图。
  - 输入 ASIN，支持按行或逗号分隔。
  - 设置 marketplace，默认 `US`。
  - 设置 language，默认 `en-US`。
- 前端提交 multipart/form-data 到后端。
- 提交成功后跳转 `/tasks/:taskId/export`。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端 API 封装。
- 创建任务页。
- 基础上传校验和错误提示。
- 任务列表页创建入口。

本阶段不做：

- 不做上传进度条。
- 不做产品图片预览编辑。
- 不做类目选择，仍使用后端默认 `CAR_STEREO`。
- 不做断点续传。
- 不做真实登录态。
