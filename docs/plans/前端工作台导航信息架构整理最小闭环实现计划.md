# 前端工作台导航信息架构整理最小闭环实现计划

> 本计划承接前端归档导出体验增强最小闭环。目标是整理运营工作台的全局导航和任务详情页入口，让已经实现的页面更容易发现和使用。

## 1. 当前基线

已完成：

- 顶部导航已有任务列表和操作审计入口。
- 任务详情页已有任务状态、输入信息、Brief 摘要、最终版本信息。
- 任务详情页可以进入竞品、Brief、文案、图片、合规、终审和导出页面。

当前不足：

- 顶部导航缺少创建任务入口。
- 任务详情页全部动作挤在一个工具栏里，不利于按流程操作。
- 任务详情页没有把“数据准备、生成审核、终审交付”分组展示。

## 2. 阶段目标

- 顶部导航提供：
  - 任务列表。
  - 创建任务。
  - 操作审计。
- 任务详情页操作入口按工作流分组：
  - 数据准备：竞品数据、Brief 审核。
  - 内容生成：文案版本、图片版本、合规报告。
  - 交付：终审选择、归档导出。
- 保留返回列表和刷新入口。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端导航和任务详情页入口整理。
- 不改变业务接口。
- 不新增页面。

本阶段不做：

- 不做权限菜单。
- 不做侧边栏重构。
- 不做面包屑组件。
- 不做视觉大改版。
