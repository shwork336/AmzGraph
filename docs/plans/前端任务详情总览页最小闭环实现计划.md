# 前端任务详情总览页最小闭环实现计划

> 本计划承接前端创建任务页最小闭环。目标是补齐运营工作台中的任务详情总览页，让运营能从一个页面查看任务主状态、输入信息、最新 Brief 摘要和后续操作入口。

## 1. 当前基线

已完成：

- 前端任务列表页 `/tasks`。
- 前端创建任务页 `/tasks/new`。
- 前端归档导出页 `/tasks/:taskId/export`。
- 后端任务详情接口 `GET /api/v1/listing/{taskId}`。
- 任务详情响应包含任务状态、类目、站点、语言、原始产品图、竞品 ASIN、最终版本 ID 和最新 Brief 摘要。

当前不足：

- 前端没有任务详情总览页。
- 任务列表只能跳到归档导出页，缺少任务整体状态入口。
- 运营无法在进入 Brief、文案、图片、合规、导出子页前先快速判断任务卡在哪个环节。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId`
- 任务列表页的任务 ID 可进入详情页。
- 任务详情总览页展示：
  - 任务主状态。
  - Brief、文案、图片子状态。
  - 类目、模板、站点、语言。
  - 原始产品图文件键列表。
  - 竞品 ASIN 列表。
  - 最新 Brief 摘要。
  - 最终文案版本 ID 和最终图片版本 ID。
- 页面提供后续操作入口：
  - 归档导出。
  - Brief 审核。
  - 文案版本。
  - 图片版本。
  - 合规报告。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端任务详情总览页。
- 前端路由和任务列表跳转调整。
- 详情页基础加载态、错误提示和刷新。
- 子页面入口按钮。

本阶段不做：

- 不实现 Brief 审核页。
- 不实现文案版本页。
- 不实现图片版本页。
- 不实现合规报告页。
- 不新增后端接口。
- 不做图片文件真实预览。
