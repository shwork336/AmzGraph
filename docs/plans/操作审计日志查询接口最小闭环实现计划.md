# 操作审计日志查询接口最小闭环实现计划

> 本计划承接操作审计日志表最小闭环。目标是把统一审计日志通过后端查询接口暴露出来，支撑后续前端审计页和问题回放。

## 1. 当前基线

已完成：

- `OperationAuditLog` 实体、仓储、服务。
- 导出取消、FAIL 合规豁免、WARNING 确认会写入统一审计日志。
- 通用 `PagedResponse` 已用于导出列表分页。

当前不足：

- 审计日志只能在服务测试中通过仓储读取。
- 没有 HTTP 查询接口。
- 后续前端审计页无法按任务、操作人、动作或目标对象筛选。

## 2. 阶段目标

- 新增 `OperationAuditLogResponse`。
- 扩展 `OperationAuditLogService` 查询能力。
- 新增 HTTP 查询接口：
  - `GET /api/v1/listing/audit-logs`
- 支持筛选：
  - `taskId`
  - `action`
  - `operatorId`
  - `targetType`
  - `targetId`
- 支持分页：
  - `page`
  - `size`
- 默认按 `createdAt` 和 `auditLogId` 倒序返回。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 查询 DTO。
- 服务层查询和分页参数校验。
- Controller 查询接口。
- 服务层和 Controller 测试。

本阶段不做：

- 不做前端审计页。
- 不做权限控制。
- 不做导出审计日志。
- 不做全文搜索。
- 不新增时间范围筛选。
