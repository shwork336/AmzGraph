# 导出文件写入幂等保护最小闭环实现计划

> 本计划承接导出取消审计最小闭环。目标是在多实例 worker、重复请求或异常重入场景下，避免同一个导出记录重复生成文件，保证导出记录的文件结果稳定可复用。

## 1. 当前基线

已完成：

- 导出记录支持 `PENDING -> RUNNING -> SUCCEEDED/FAILED`。
- worker 通过条件更新领取锁完成 `PENDING -> RUNNING`。
- 导出记录已有 `fileUrl`、`manifestUrl`、`startedAt`、`updatedAt`。
- 同步运行接口会先领取 `PENDING` 再执行。
- worker 领取失败时返回 `null` 并跳过。

当前不足：

- 如果执行阶段重复进入同一条已成功记录，服务没有明确的幂等返回边界。
- 如果 `RUNNING` 记录已经写入文件路径但状态重入，可能再次写入文件。
- 文件写入前没有集中校验“该导出记录是否已有可复用输出”。

## 2. 阶段目标

- 在导出执行入口增加幂等保护：
  - `SUCCEEDED` 且已有导出文件路径时直接返回当前记录。
  - `RUNNING` 且已有导出文件路径时补正为 `SUCCEEDED` 并返回。
  - `RUNNING` 且没有完整输出路径时不通过公开运行接口重复执行。
  - 其他状态不允许执行。
- 文件输出路径是否完整按导出格式判断：
  - ZIP 需要 `fileUrl` 和 `manifestUrl`。
  - Markdown、Excel、Word 只需要 `fileUrl`。
- 重复运行不会再次写文件。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 修改 `ExportPackageService` 的执行阶段幂等判断。
- 补充服务层测试，验证已成功记录不会重复写文件。
- 补充 RUNNING 已有输出路径补正测试。
- 更新计划验收状态。

本阶段不做：

- 不改 `FileStoragePort`。
- 不引入分布式锁、中间件或对象存储条件写。
- 不删除历史重复文件。
- 不改变现有导出接口路径。
- 不改变 worker 批处理统计语义。
