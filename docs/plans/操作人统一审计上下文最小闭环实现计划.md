# 操作人统一审计上下文最小闭环实现计划

> 本计划承接导出文件写入幂等保护最小闭环。目标是在没有完整权限系统前，先统一运营/管理员操作人的传入边界，为后续登录态、权限和审计日志接入预留稳定接口。

## 1. 当前基线

已完成：

- 导出取消记录 `canceledBy`、`cancelReason`、`canceledAt`。
- FAIL 合规豁免记录 `complianceReviewedBy`、`complianceReviewReason`、`complianceReviewedAt`。
- WARNING 人工确认复用同一套合规审计字段。
- 相关接口都要求请求体传操作人。

当前不足：

- 操作人字段散落在不同请求体字段中。
- 前端后续接入登录态后，无法通过统一 header 传递当前操作人。
- Controller 没有统一的“请求操作人解析”边界。

## 2. 阶段目标

- 新增统一操作人请求头：
  - `X-Operator-Id`
- Controller 解析操作人时：
  - 优先使用 `X-Operator-Id`。
  - header 为空时回退请求体字段。
- 导出取消：
  - header 优先覆盖 `canceledBy`。
  - `cancelReason` 仍来自 body。
- 合规豁免和 WARNING 确认：
  - header 优先覆盖 `reviewedBy`。
  - `reason` 仍来自 body。
- 保留原有 body 操作人字段，兼容当前调用方。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 新增轻量操作人解析工具。
- 修改 Controller，在进入应用服务前合并 header 和 body。
- 保留服务层原有必填防御。
- 补充 Controller 测试。

本阶段不做：

- 不新增登录、权限或用户表。
- 不引入 Spring Security。
- 不修改数据库结构。
- 不新增独立审计日志表。
- 不移除请求体中的操作人字段。
