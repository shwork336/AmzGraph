# 前端竞品数据页最小闭环实现计划

> 本计划承接前端操作审计总览页最小闭环。目标是补齐运营工作台中的竞品数据页，让运营能查看任务竞品快照历史、每个 ASIN 最新快照，并手工批量补录竞品数据。

## 1. 当前基线

已完成：

- 后端手工竞品补录接口 `POST /api/v1/listing/{taskId}/competitors/manual`。
- 后端竞品历史查询接口 `GET /api/v1/listing/{taskId}/competitors`。
- 后端每个 ASIN 最新快照查询接口 `GET /api/v1/listing/{taskId}/competitors/latest`。
- 前端任务详情页 `/tasks/:taskId`。

当前不足：

- 前端没有竞品数据页。
- 任务详情页没有竞品数据入口。
- 运营无法在前端查看竞品历史或补录竞品快照。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId/competitors`
- 前端 API client 支持：
  - 查询竞品历史。
  - 查询每个 ASIN 最新快照。
  - 手工批量补录竞品快照。
- 竞品数据页展示：
  - 任务当前状态。
  - latest 快照表。
  - 全部历史表。
  - 选中快照详情。
- 页面提供操作：
  - 刷新竞品数据。
  - 手工补录单条快照。
  - 返回任务详情。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端竞品数据页。
- 前端路由和任务详情页入口调整。
- 单条手工快照表单，提交时使用后端批量接口。

本阶段不做：

- 不接入 Bright Data。
- 不做多行批量编辑器。
- 不做竞品数据 diff。
- 不新增后端接口。
