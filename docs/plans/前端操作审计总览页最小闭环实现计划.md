# 前端操作审计总览页最小闭环实现计划

> 本计划承接前端合规报告页最小闭环。目标是补齐运营工作台中的操作审计总览页，让运营能跨任务查询导出取消、合规豁免、WARNING 确认等审计记录。

## 1. 当前基线

已完成：

- 后端审计日志查询接口 `GET /api/v1/listing/audit-logs`。
- 查询支持 `taskId`、`action`、`operatorId`、`targetType`、`targetId`、`page`、`size`。
- 前端归档导出页已有任务内审计日志展示。
- 前端合规报告页已支持 FAIL 豁免和 WARNING 确认。

当前不足：

- 前端没有全局操作审计总览页。
- 前端审计 API client 只封装了 `taskId` 查询。
- 页面顶部缺少基础导航入口。
- 运营无法跨任务按操作人、动作或目标对象追踪操作记录。

## 2. 阶段目标

- 新增路由：
  - `/audit-logs`
- 前端 API client 支持完整审计筛选参数。
- 操作审计页展示：
  - 审计日志表格。
  - 动作、操作人、目标类型、目标 ID、任务 ID、原因、详情 JSON、创建时间。
  - 分页。
- 页面提供筛选：
  - 任务 ID。
  - 动作。
  - 操作人。
  - 目标类型。
  - 目标 ID。
- App 顶部提供基础导航：
  - 任务列表。
  - 操作审计。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端操作审计总览页。
- 前端审计查询 client 参数扩展。
- App 顶部基础导航。

本阶段不做：

- 不新增后端接口。
- 不做审计日志导出。
- 不做复杂 JSON 格式化编辑器。
- 不做权限系统。
