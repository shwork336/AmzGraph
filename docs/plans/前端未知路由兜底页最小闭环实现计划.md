# 前端未知路由兜底页最小闭环实现计划

> 本计划承接前端状态标签样式统一。目标是为工作台补齐未知路径兜底页，避免用户访问错误 URL 时内容区域空白。

## 1. 当前基线

已完成：

- 前端工作台已覆盖任务列表、创建任务、任务详情、竞品、Brief、文案、图片、合规、终审、归档导出、操作审计等页面。
- 路由已经按需拆包。
- 一级路由和任务子路由已完成浏览器冒烟验证。

当前不足：

- `router.ts` 没有 catch-all 路由。
- 访问未知路径时只保留顶部导航，内容区没有明确错误提示。

## 2. 阶段目标

- 新增未知路由兜底页面。
- 未知路径展示当前路径和返回入口。
- 保持现有路由不变。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 新增 `NotFoundPage.vue`。
- 在 `router.ts` 注册 `/:pathMatch(.*)*`。
- 保持路由按需加载。

本阶段不做：

- 不改后端静态资源部署配置。
- 不新增鉴权或权限路由。
- 不改变已有页面导航。

## 4. 任务拆解

- [x] 新增未知路由兜底页。
- [x] 注册 catch-all 路由。
- [x] 前端 `npm.cmd run build` 通过。
- [x] 后端 `.\gradlew.bat test --console=plain` 通过。

## 5. 验收清单

- [x] 未知路径不再出现内容区空白。
- [x] 用户可以从兜底页返回任务列表或创建任务。
- [x] 前端构建通过。
- [x] 后端完整测试通过。
