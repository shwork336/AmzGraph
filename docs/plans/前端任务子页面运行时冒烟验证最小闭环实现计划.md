# 前端任务子页面运行时冒烟验证最小闭环实现计划

> 本计划承接前端运行时冒烟验证最小闭环。目标是在一级路由验证之外，覆盖所有带 `taskId` 的工作流子页面，确认它们在真实浏览器中可以完成 Vue 挂载，并在后端无数据或未启动时展示页面结构和错误态。

## 1. 当前基线

已完成：

- `/tasks`、`/tasks/new`、`/audit-logs` 已通过浏览器冒烟。
- 根级 `n-message-provider` 已补齐，`useMessage()` 页面不再因为缺失上下文而挂载失败。
- 前端构建和后端完整测试已通过。

当前不足：

- 带 `taskId` 的工作流子页面尚未逐个用浏览器打开验证。
- 这些页面覆盖竞品、Brief、文案、图片、合规、终审、归档导出等主要操作入口，运行时风险更高。

## 2. 阶段目标

- 所有任务子页面路由可以在浏览器中正常挂载。
- 后端不可用或占位 `taskId` 无数据时，页面保留导航、工具栏、错误态或空状态，不发生白屏。
- 控制台无阻断性 Vue error。

## 3. 范围边界

本阶段做：

- 使用占位任务 ID 打开任务详情和全部子流程页面。
- 检查页面标题、关键按钮或错误态是否可见。
- 检查浏览器控制台错误。
- 根据发现修复前端运行时基础问题。

本阶段不做：

- 不准备后端测试数据。
- 不验证提交、生成、审核、导出等真实业务动作。
- 不修改后端业务逻辑。
