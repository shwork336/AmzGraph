# 前端图片版本页最小闭环实现计划

> 本计划承接前端文案版本页最小闭环。目标是补齐运营工作台中的图片版本页，让运营能在 Brief 批准后显式生成占位图片版本，并查看图片资产、尺寸和合规状态。

## 1. 当前基线

已完成：

- 后端图片生成接口 `POST /api/v1/listing/{taskId}/versions/image/generate`。
- 后端图片版本列表接口 `GET /api/v1/listing/{taskId}/versions/image`。
- 后端图片资产列表接口 `GET /api/v1/listing/{taskId}/versions/image/{imageVersionId}/assets`。
- 前端任务详情页 `/tasks/:taskId`。
- 前端文案版本页 `/tasks/:taskId/text`。

当前不足：

- 前端没有图片版本页。
- 任务详情页的图片版本入口仍是占位提示。
- 运营无法在前端显式触发图片生成，也无法查看图片资产类型、Prompt、生成 URL、尺寸和合规状态。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId/images`
- 前端 API client 支持：
  - 触发初始图片版本生成。
  - 查询图片版本列表。
  - 查询图片版本下的图片资产。
- 图片版本页展示：
  - 任务当前状态。
  - 图片版本历史。
  - 当前图片版本资产列表。
  - 资产类型、Prompt、生成 URL、尺寸、合规状态、合规方法、合规问题。
- 页面提供操作：
  - 生成图片版本。
  - 刷新版本和资产。
  - 从版本历史中查看任意版本资产。
  - 返回任务详情。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端图片版本页。
- 前端路由和任务详情页入口调整。
- 图片版本和资产查询、图片生成的基础加载态和错误提示。

本阶段不做：

- 不实现真实图片预览鉴权。
- 不实现图片人工迭代接口。
- 不实现合规人工确认入口。
- 不实现最终图片选择。
- 不新增后端接口。
