# 前端子页面显示工具统一最小闭环实现计划

> 本计划承接前端操作人状态共享。目标是把任务子页面中重复的时间显示、空值显示和错误消息提取逻辑统一到 `frontend/src/utils/display.ts`，减少页面之间文案和格式不一致。

## 1. 当前基线

已完成：

- `displayTime()`、`displayValue()`、`errorMessage()` 已在共享工具中存在。
- 任务列表、任务详情、归档导出、操作审计等页面已经部分使用共享显示工具。
- 前端工作台主要页面已完成浏览器冒烟。

当前不足：

- Brief、竞品、文案、图片、合规、终审等子页面仍有局部 `displayTime()` / `displayValue()`。
- 多个页面仍直接写 `error instanceof Error ? error.message : fallback`。
- 显示格式和错误兜底逻辑分散，不利于后续统一调整。

## 2. 阶段目标

- 子页面复用 `displayTime()`、`displayValue()` 和 `errorMessage()`。
- 移除重复局部显示函数。
- 页面 UI 和接口调用行为保持不变。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 替换子页面重复显示工具逻辑。
- 保持现有模板结构和交互不变。
- 不新增业务页面。

本阶段不做：

- 不改状态标签映射。
- 不改后端 API。
- 不引入 i18n。
