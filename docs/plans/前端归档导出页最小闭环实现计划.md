# 前端归档导出页最小闭环实现计划

> 本计划承接后端导出生命周期和操作审计查询闭环。目标是创建第一版 Vue 3 内部运营工作台前端，并优先落地归档导出页，让运营可以围绕单个 Listing 任务查看最终选择、管理导出记录和查看审计日志。

## 1. 当前基线

已完成：

- 后端提供任务详情查询。
- 后端提供导出记录创建、列表分页筛选、运行、取消、重试。
- 后端提供操作审计日志查询。
- 架构文档规划前端技术栈为 Vue 3、TypeScript、Vite、Naive UI。

当前不足：

- 仓库尚无前端工程。
- 归档导出页还不能通过 UI 调用后端接口。
- 操作人统一 header 需要前端固定传入。

## 2. 阶段目标

- 新建前端工程目录 `frontend`。
- 使用 Vue 3 + TypeScript + Vite + Naive UI。
- 实现路由：
  - `/tasks/:taskId/export`
  - `/` 默认跳转或承载快速入口
- 归档导出页支持：
  - 输入或读取 `taskId`。
  - 查询任务详情，展示最终文案版本和最终图片版本。
  - 查询导出历史，支持格式、状态筛选和分页。
  - 创建待执行导出记录。
  - 运行 PENDING 导出。
  - 取消 PENDING 导出并填写取消原因。
  - 重试 FAILED 导出。
  - 查询任务维度审计日志。
- 前端构建通过。

## 3. 范围边界

本阶段做：

- 前端工程骨架。
- 归档导出页。
- 最小 API 封装。
- 基础错误提示和加载态。
- 构建验证。

本阶段不做：

- 不接登录系统。
- 不做完整任务列表。
- 不做创建任务页。
- 不做 Brief、文案、图片和合规完整页面。
- 不做端到端浏览器测试。
