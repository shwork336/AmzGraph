# 导出列表查询最小闭环实现计划

> 本计划承接 ZIP、Markdown、Excel、Word 导出与失败重试闭环。
> 本阶段目标是为前端归档导出页提供任务维度的导出历史列表，方便运营查看每次导出的格式、状态、文件 URL 和失败原因。

## 1. 当前基线

已完成：

- 已完成任务可导出 ZIP、Markdown、Excel、Word。
- 四种导出格式都支持失败重试。
- 单个导出包可通过 `GET /api/v1/listing/export/{exportPackageId}` 查询。
- `ExportPackageRepository` 已有按任务倒序查询方法：
  - `findByTaskIdOrderByCreatedAtDescExportPackageIdDesc(String taskId)`

当前未完成：

- 没有按任务查询导出历史的服务方法。
- 没有按任务查询导出历史的 HTTP 接口。
- 前端归档导出页无法一次性展示任务下全部导出记录。

## 2. 阶段目标

本阶段完成后，应满足：

- 可按任务 ID 查询该任务下全部导出记录。
- 列表按创建时间倒序、导出包 ID 倒序返回。
- 响应继续使用 `ExportPackageResponse`，不暴露 JPA 实体。
- 不存在任务返回 `TASK_NOT_FOUND`。
- 没有导出记录时返回空数组。
- 查询接口不触发导出、重试或任务状态变更。
- 完整测试通过。

## 3. 范围边界

### 3.1 本阶段做

- 扩展 `ExportPackageService`：
  - 新增 `listExportPackages(String taskId)`
  - 先校验任务存在
  - 读取仓储列表并转换为 DTO
- 扩展 `ListingTaskController`：
  - 新增 `GET /api/v1/listing/{taskId}/exports`
- 补充服务测试：
  - 返回指定任务下所有导出记录
  - 按倒序返回
  - 空列表返回空数组
  - 任务不存在返回 `TASK_NOT_FOUND`
- 补充 Controller 测试：
  - 成功返回统一 `ApiResponse`
  - 空列表返回 `[]`
  - 业务错误透传稳定错误码

### 3.2 本阶段不做

- 不做分页。
- 不做格式筛选。
- 不做状态筛选。
- 不做下载代理。
- 不做文件 URL 签名。
- 不改造为异步导出任务列表。

## 4. 设计原则

- 查询只读，不改变任务、导出包或文件状态。
- 任务不存在和任务存在但无导出记录要区分：前者报错，后者返回空列表。
- DTO 继续复用 `ExportPackageResponse`，保持单条查询和列表查询一致。
- 当前阶段先用全量列表，后续导出记录增长后再加分页。
