# 前端状态标签样式统一最小闭环实现计划

> 本计划承接前端子页面显示工具统一。目标是把前端页面中重复的状态到 `n-tag` 类型映射收敛为共享工具，避免任务、生成、合规、导出等状态在不同页面呈现不一致。

## 1. 当前基线

已完成：

- 前端显示时间、空值、错误消息已经统一到 `display.ts`。
- 工作台各页面已使用 `n-tag` 展示任务、生成、合规、导出等状态。
- 前端构建和后端测试持续通过。

当前不足：

- 多个页面重复定义 `statusType()`。
- 同一状态在不同页面的映射条件散落，例如 `WAIT`、`GENERATING`、`RUNNING`、`FAILED`、`PASS`、`WARNING`。
- 后续新增状态时需要逐页查找修改。

## 2. 阶段目标

- 新增共享状态样式工具。
- 页面复用共享工具计算 `n-tag` 类型。
- 保持当前状态文本和页面布局不变。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 新增 `frontend/src/utils/status.ts`。
- 替换页面内重复 `statusType()`。
- 不改变业务状态值和后端接口。

本阶段不做：

- 不做 i18n 状态文案映射。
- 不调整表格列和工作流交互。
- 不修改后端枚举。

## 4. 任务拆解

- [x] 新增共享状态样式工具。
- [x] 任务列表和任务详情页使用共享状态样式。
- [x] 文案、图片、合规、终审页使用共享状态样式。
- [x] 归档导出页使用共享状态样式。
- [x] 前端 `npm.cmd run build` 通过。
- [x] 后端 `.\gradlew.bat test --console=plain` 通过。

## 5. 验收清单

- [x] 状态标签颜色映射行为保持兼容。
- [x] 前端构建通过。
- [x] 后端完整测试通过。
