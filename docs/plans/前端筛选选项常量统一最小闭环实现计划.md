# 前端筛选选项常量统一最小闭环实现计划

> 本计划承接前端任务路由路径工具统一。目标是把任务列表、导出列表和操作审计中的稳定筛选选项收敛到共享常量，减少页面内重复枚举。

## 1. 当前基线

已完成：

- 前端 API、显示工具、状态样式、任务路由路径已完成统一。
- 任务列表、导出页、操作审计页均支持筛选。

当前不足：

- 任务状态、站点、类目、导出格式、导出状态、审计动作、审计目标类型等选项直接写在页面内。
- 后续新增或调整选项时需要逐页修改。

## 2. 阶段目标

- 新增共享筛选选项常量。
- 任务列表页、归档导出页、操作审计页使用共享选项。
- 保持现有筛选值和显示文案不变。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 新增 `frontend/src/utils/options.ts`。
- 替换页面内稳定枚举选项。

本阶段不做：

- 不接后端动态字典。
- 不新增 i18n。
- 不改变筛选条件语义。

## 4. 任务拆解

- [x] 新增共享筛选选项常量。
- [x] 任务列表页使用共享选项。
- [x] 归档导出页使用共享选项。
- [x] 操作审计页使用共享选项。

- [x] 后端 `.\gradlew.bat test --console=plain` 通过。

## 5. 验收清单

- [x] 现有选项 label/value 保持不变。
- [x] 前端构建通过。
