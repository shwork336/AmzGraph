# 导出时间字段增强最小闭环实现计划

> 本计划承接导出 Worker 领取锁最小闭环。目标是给导出记录补齐 `startedAt` 和 `updatedAt`，让执行追踪和超时恢复不再只依赖 `createdAt`。

## 1. 当前基线

已完成：

- 导出记录有 `createdAt`。
- worker 支持超时恢复，但当前用 `createdAt` 判断超时。
- worker 支持原子领取 `PENDING -> RUNNING`。
- 导出响应返回 `createdAt`。

当前不足：

- 没有记录导出实际开始时间。
- 没有记录状态最后更新时间。
- `RUNNING` 超时用 `createdAt` 判断，不够准确。
- API 响应无法展示 started/updated 时间。

## 2. 阶段目标

- `ExportPackage` 新增：
  - `startedAt`
  - `updatedAt`
- 新建记录时：
  - `createdAt` 和 `updatedAt` 自动填充。
  - 立即导出和 worker 领取进入 `RUNNING` 时填充 `startedAt`。
- 状态变更时更新 `updatedAt`。
- 超时恢复基于 `updatedAt` 查询。
- `ExportPackageResponse` 返回 `startedAt` 和 `updatedAt`。
- 完整测试通过。

## 3. 范围边界

本阶段做：

- 修改实体字段。
- 修改响应 DTO。
- 修改仓储超时查询。
- 修改导出服务状态更新时间维护。
- 修改测试构造器和断言。

本阶段不做：

- 不新增数据库 migration 文件。
- 不新增锁持有人字段。
- 不新增取消人/取消原因字段。
- 不改变现有导出接口路径。
