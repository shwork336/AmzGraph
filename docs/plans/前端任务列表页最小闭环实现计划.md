# 前端任务列表页最小闭环实现计划

> 本计划承接前端归档导出页最小闭环。目标是补齐运营工作台入口页，让运营能从任务列表查看任务状态，并进入归档导出页。

## 1. 当前基线

已完成：

- 前端工程 `frontend`。
- 归档导出页 `/tasks/:taskId/export`。
- 后端任务详情查询 `GET /api/v1/listing/{taskId}`。
- 后端导出和审计查询能力。

当前不足：

- 后端没有任务列表查询接口。
- 前端首页只能手动输入任务 ID。
- 运营无法从 UI 扫描任务状态并进入归档导出。

## 2. 阶段目标

- 后端新增任务列表查询：
  - `GET /api/v1/listing/tasks`
- 支持筛选：
  - `status`
  - `marketplace`
  - `categoryCode`
- 支持分页：
  - `page`
  - `size`
- 默认按 `updatedAt` 和 `taskId` 倒序返回。
- 前端新增 `/tasks` 页面。
- 首页默认跳转到 `/tasks`。
- 任务列表页展示任务主状态、子流程状态、类目、站点、最终版本、更新时间。
- 任务列表页可跳转到归档导出页。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 后端任务列表 DTO、服务查询、Controller 接口。
- 前端任务列表 API、路由和页面。
- 基础筛选、分页、跳转。

本阶段不做：

- 不做创建任务页。
- 不做任务详情总览页。
- 不做任务删除或批量操作。
- 不做权限控制。
- 不做服务端复杂组合搜索。
