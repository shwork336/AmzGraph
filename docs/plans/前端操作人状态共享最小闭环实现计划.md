# 前端操作人状态共享最小闭环实现计划

> 本计划承接前端总览页错误态持久化。目标是把人工审核、补录、合规、导出等页面使用的操作人 ID 读写逻辑收敛到共享工具，减少审计操作入口之间的默认值和持久化行为不一致。

## 1. 当前基线

已完成：

- Brief 审核、竞品补录、合规审核、归档导出等页面已经支持填写操作人。
- 操作人会写入 `localStorage`，用于后续审计操作。
- 后端已有操作审计日志查询和前端操作审计总览页。

当前不足：

- 多个页面直接使用 `localStorage.getItem('amzgraph.operatorId') ?? 'operator@example.com'`。
- 多个页面分别调用 `localStorage.setItem()`，trim 和默认值规则散落。
- 后续新增人工操作页面时容易复制出不一致行为。

## 2. 阶段目标

- 提供共享的操作人状态工具。
- 保持现有 UI 和接口调用行为不变。
- 页面只通过共享工具读取默认操作人和保存操作人。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 新增 `frontend/src/utils/operator.ts`。
- 替换当前人工操作页面里的直接 `localStorage` 读写。
- 不改变 localStorage key，保留既有用户本地状态。

本阶段不做：

- 不引入登录/权限系统。
- 不修改后端审计上下文。
- 不新增全局用户设置页面。
