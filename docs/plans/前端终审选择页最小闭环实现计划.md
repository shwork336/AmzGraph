# 前端终审选择页最小闭环实现计划

> 本计划承接前端图片版本页最小闭环。目标是补齐运营工作台中的终审选择页，让运营能选择最终文案版本和图片版本，并把任务推进到 `COMPLETED`。

## 1. 当前基线

已完成：

- 后端终审选择接口 `POST /api/v1/listing/{taskId}/final/approve`。
- 后端文案版本列表接口 `GET /api/v1/listing/{taskId}/versions/text`。
- 后端图片版本列表接口 `GET /api/v1/listing/{taskId}/versions/image`。
- 前端任务详情页 `/tasks/:taskId`。
- 前端文案版本页 `/tasks/:taskId/text`。
- 前端图片版本页 `/tasks/:taskId/images`。

当前不足：

- 前端没有终审选择页。
- 任务详情页没有终审入口。
- 运营无法在前端选择最终文案版本和图片版本完成任务。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId/final`
- 前端 API client 支持：
  - 提交终审选择。
- 终审选择页展示：
  - 任务当前状态。
  - 可选文案版本列表。
  - 可选图片版本列表。
  - 当前选择摘要。
- 页面提供操作：
  - 选择文案版本。
  - 选择图片版本。
  - 提交终审。
  - 返回任务详情。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端终审选择页。
- 前端路由和任务详情页入口调整。
- 终审选择提交的基础加载态和错误提示。

本阶段不做：

- 不实现文案或图片 diff。
- 不实现图片资产逐张终审。
- 不新增后端接口。
- 不自动生成导出包。
