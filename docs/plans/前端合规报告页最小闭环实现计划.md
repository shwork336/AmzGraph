# 前端合规报告页最小闭环实现计划

> 本计划承接前端终审选择页最小闭环。目标是补齐运营工作台中的图片资产合规报告页，让运营能按图片版本查看资产合规状态，并对 `FAIL` 资产提交管理员豁免、对 `WARNING` 资产提交人工确认。

## 1. 当前基线

已完成：

- 后端图片版本列表接口 `GET /api/v1/listing/{taskId}/versions/image`。
- 后端图片资产列表接口 `GET /api/v1/listing/{taskId}/versions/image/{imageVersionId}/assets`。
- 后端 FAIL 豁免接口 `POST /api/v1/listing/{taskId}/versions/image/{imageVersionId}/assets/{assetId}/compliance/approve`。
- 后端 WARNING 确认接口 `POST /api/v1/listing/{taskId}/versions/image/{imageVersionId}/assets/{assetId}/compliance/confirm-warning`。
- 前端任务详情页 `/tasks/:taskId`。
- 前端图片版本页 `/tasks/:taskId/images`。

当前不足：

- 前端没有独立合规报告页。
- 任务详情页的合规报告入口仍是占位提示。
- 运营无法在前端集中查看资产合规状态，也无法提交豁免或确认原因。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId/compliance`
- 前端 API client 支持：
  - FAIL 图片资产合规豁免。
  - WARNING 图片资产人工确认。
- 合规报告页展示：
  - 任务当前状态。
  - 图片版本列表。
  - 当前版本资产合规列表。
  - 资产 Prompt、生成 URL、尺寸、合规状态、合规方法、合规问题、确认/豁免信息。
- 页面提供操作：
  - 切换图片版本。
  - 对 `FAIL` 资产提交管理员豁免原因。
  - 对 `WARNING` 资产提交人工确认原因。
  - 返回任务详情。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端合规报告页。
- 前端路由和任务详情页入口调整。
- 单个资产的 FAIL 豁免和 WARNING 确认。

本阶段不做：

- 不实现权限系统。
- 不实现批量豁免或批量确认。
- 不实现撤销豁免或撤销确认。
- 不新增后端接口。
- 不修改导出策略。
