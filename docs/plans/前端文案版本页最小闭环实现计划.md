# 前端文案版本页最小闭环实现计划

> 本计划承接前端 Brief 审核页最小闭环。目标是补齐运营工作台中的文案版本页，让运营能在 Brief 批准后显式生成占位文案版本，并查看全部文案版本内容。

## 1. 当前基线

已完成：

- 后端文案生成接口 `POST /api/v1/listing/{taskId}/versions/text/generate`。
- 后端文案版本列表接口 `GET /api/v1/listing/{taskId}/versions/text`。
- 前端任务详情页 `/tasks/:taskId`。
- 前端 Brief 审核页 `/tasks/:taskId/brief`。

当前不足：

- 前端没有文案版本页。
- 任务详情页的文案版本入口仍是占位提示。
- 运营无法在前端显式触发文案生成，也无法查看标题、五点、描述、关键词和合规警告。

## 2. 阶段目标

- 新增路由：
  - `/tasks/:taskId/text`
- 前端 API client 支持：
  - 触发初始文案生成。
  - 查询文案版本列表。
- 文案版本页展示：
  - 任务当前状态。
  - 文案版本历史。
  - 选中文案版本详情。
  - 标题、五点、描述、后台搜索词、目标关键词、合规警告、质量评分。
- 页面提供操作：
  - 生成文案。
  - 刷新版本列表。
  - 从版本历史中查看任意版本。
  - 返回任务详情。
- 前端构建和后端测试通过。

## 3. 范围边界

本阶段做：

- 前端文案版本页。
- 前端路由和任务详情页入口调整。
- 文案版本查询和生成的基础加载态、错误提示。

本阶段不做：

- 不实现文案人工迭代接口。
- 不实现最终文案选择。
- 不实现文案 diff。
- 不新增后端接口。
- 不触发图片生成。
