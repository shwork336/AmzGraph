# 前端总览页错误态持久化最小闭环实现计划

> 本计划承接前端 API 错误解析健壮化。目标是让任务列表和操作审计两个总览入口在接口加载失败时，保留页面级错误提示，避免只弹出短暂 toast 后页面看起来像“暂无数据”。

## 1. 当前基线

已完成：

- API 客户端已兼容 HTTP 错误、非 JSON 响应和空响应。
- 多数任务子页面已有 `errorText` + `n-alert` 页面级错误态。
- 任务列表和操作审计页已有空状态描述。

当前不足：

- 任务列表加载失败时只调用 `message.error()`，页面级没有保留失败原因。
- 操作审计加载失败时只调用 `message.error()`，容易与“暂无审计日志”混淆。

## 2. 阶段目标

- 任务列表页加载失败时显示持久错误提示。
- 操作审计页加载失败时显示持久错误提示。
- 下一次成功加载或重新查询前清理旧错误。
- 不改变现有筛选、分页和表格交互。

## 3. 范围边界

本阶段做：

- 增加两个总览页的 `errorText` 状态。
- 在页面工具栏下方展示 `n-alert`。
- 前端构建和后端测试回归。

本阶段不做：

- 不改后端接口。
- 不引入全局错误边界。
- 不新增前端测试框架。
