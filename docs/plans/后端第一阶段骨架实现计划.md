# 后端第一阶段骨架实现计划

> **面向 AI 代理的工作者：** 建议使用 subagent-driven-development 或 executing-plans 按任务执行此计划。步骤使用复选框语法跟踪进度。每个任务完成后运行对应测试，再进入下一任务。

**目标：** 在现有 Spring Boot 项目中落地模块化单体后端骨架，接入 PostgreSQL，建立核心领域模型，并跑通“创建任务 -> 上传 Markdown 和产品图 -> 保存任务 -> 读取 Car Stereo 模板 -> 返回任务详情”的最小闭环。

**架构：** 后端继续作为一个 Spring Boot 单体应用部署，内部按 `listing`、`template`、`common` 等业务模块拆包。第一阶段只实现稳定骨架和任务提交闭环，不接真实 Bright Data、真实 AI 文案生成、真实图片生成和导出。

**技术栈：** Spring Boot 4、Spring Web MVC、Spring Data JPA、PostgreSQL、Lombok、JUnit。

---

## 1. 范围边界

### 1.1 本阶段要完成

- 规范包名为 `com.snails.ecommerce`。
- 配置 PostgreSQL 数据源和 JPA 基础配置。
- 建立统一响应、统一异常、基础文件校验。
- 建立本地文件存储端口和本地实现。
- 建立第一批核心 JPA 实体和枚举。
- 内置 `CAR_STEREO` 类目模板。
- 实现任务提交接口。
- 实现任务详情查询接口。
- 使用占位 Brief 生成结果支撑最小闭环。
- 添加覆盖核心服务和接口的测试。

### 1.2 本阶段不做

- 不接真实 Bright Data。
- 不接真实 Spring AI Alibaba 模型调用。
- 不生成真实图片资产。
- 不做 ZIP / Excel / Word 导出。
- 不做前端页面。
- 不做复杂权限系统。
- 不引入 MQ 或 BPMN 工作流引擎。
