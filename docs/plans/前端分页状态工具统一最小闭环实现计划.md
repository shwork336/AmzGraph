# 前端分页状态工具统一最小闭环实现计划

> 本计划承接前端筛选选项常量统一。目标是把列表页中重复的分页状态管理收敛为共享工具，减少 `page/size/totalItems/totalPages` 和 page-size 重置逻辑重复。

## 1. 当前基线

已完成：

- 任务列表、导出页、操作审计页均支持分页。
- 筛选选项已经统一到共享常量。
- 前端构建和后端测试持续通过。

当前不足：

- 多个页面重复定义分页状态。
- 多个页面重复写 `filters.page = 1; load...()`。
- 响应分页字段同步逻辑散落在页面内。

## 2. 阶段目标

- 新增共享分页状态工具。
- 任务列表页和操作审计页使用共享分页状态。
- 归档导出页的导出列表和审计列表使用共享分页状态。
- 保持现有分页 UI 和请求参数不变。

## 3. 范围边界

本阶段做：

- 新增 `frontend/src/utils/pagination.ts`。
- 替换页面内分页状态管理。

本阶段不做：

- 不改后端分页协议。
- 不改分页组件样式。
- 不引入表格封装组件。
